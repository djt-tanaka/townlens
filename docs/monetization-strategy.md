# マネタイズ戦略書

## エグゼクティブサマリー

estat-city-report は、政府統計（e-Stat）と不動産情報ライブラリAPIを統合し、市区町村・メッシュ・駅圏の多角的比較レポートを自動生成するプロダクトである。現在CLIツールとして高い完成度を持つが、ビジネスとしてのマネタイズには**プロダクト形態の転換**と**ターゲット市場の明確化**が必要となる。

本戦略では、3つのフェーズに分けて段階的にマネタイズを実現するロードマップを提案する。

---

## 1. 市場分析

### 1.1 ターゲット市場

| セグメント | 市場規模感 | ペインポイント | 支払意思 |
|-----------|-----------|--------------|---------|
| **引越し検討者（個人）** | 年間500万件以上の引越し | 街選びの判断材料が散在・比較が困難 | 低〜中（月額500〜2,000円） |
| **不動産仲介業者** | 約12万事業所 | 顧客への提案資料作成が手作業 | 中〜高（月額5,000〜30,000円） |
| **不動産デベロッパー** | 大手〜中堅100社超 | 用地選定・マーケティングに統計分析が必要 | 高（月額50,000〜300,000円） |
| **自治体・シンクタンク** | 約1,700自治体 | 移住促進・政策立案の根拠データ | 高（年額100万円〜） |
| **メディア・ライター** | 不動産系メディア多数 | 街ランキング記事のデータ根拠 | 中（記事単位 or 月額） |

### 1.2 競合分析

| 競合 | 強み | 弱み | 当プロダクトの差別化 |
|------|------|------|-------------------|
| SUUMO 街診断 | 圧倒的ブランド・集客力 | 主観的口コミ中心、定量データ不足 | **政府統計ベースの客観スコアリング** |
| HOME'S まちむす | UI/UX の良さ | 比較機能が限定的 | **多指標×重み付けの柔軟な比較** |
| TownU (三井不動産) | 大手のリソース | 自社物件への誘導バイアス | **中立的・第三者的な分析** |
| 各自治体オープンデータ | 無料・一次データ | 散在・フォーマット不統一・比較不能 | **統合・正規化・レポート自動生成** |

### 1.3 当プロダクトの核心的価値（モート）

1. **データ統合力**: e-Stat + 不動産情報ライブラリ + 災害データを1つのスコアに統合
2. **スコアリングエンジン**: 正規化・パーセンタイル・重みプリセット・信頼度評価の独自ロジック
3. **粒度の柔軟性**: 市区町村 / 1kmメッシュ / 駅圏 の3階層で分析可能
4. **レポート自動生成**: プロフェッショナルなPDFを完全自動生成

**目標**: ユーザー基盤の構築と市場検証

├── 月3件まで市区町村比較（2都市まで）
├── 基本スコア表示（総合スコアのみ）
├── 簡易Webレポート
└── 広告表示あり

スタンダードプラン（¥980/月）
├── 無制限の市区町村比較（5都市まで）
├── 全指標の詳細スコア + ダッシュボード
├── PDF レポートダウンロード
├── メッシュ分析（月5件）
├── 駅圏分析（月5件）
├── 重みカスタマイズ
└── 広告非表示

プレミアムプラン（¥2,980/月）
├── 無制限の全分析
├── 全3モード無制限
├── カスタム重みプリセット保存
├── 時系列比較（経年変化）
├── APIアクセス（月1,000リクエスト）

## 5. 戦略的優先度マトリクス

```
            高収益ポテンシャル
                 │
    [自治体PKG]  │  [B2B SaaS]
                 │
低リスク ────────┼──────── 高リスク
                 │
    [B2Cフリー   │  [APIプラット
     ミアム]     │   フォーム]
                 │
            低収益ポテンシャル
```

**推奨優先順位**:
1. **B2C フリーミアム**（低リスク・市場検証）→ まず始めるべき
2. **B2B SaaS**（高収益・明確なペイン）→ 最大の収益ドライバー
3. **自治体パッケージ**（高収益・長期契約）→ 安定収益源
4. **API プラットフォーム**（高リスク・高リターン）→ スケール手段

## 6. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| e-Stat API の利用制限変更 | 高 | データのローカルキャッシュ強化、バルクダウンロード対応 |
| 大手プレイヤーの参入 | 中 | スコアリングエンジンの精度・柔軟性で差別化、ニッチ市場の深耕 |
| データの鮮度・正確性 | 高 | 信頼度スコアの透明化、複数データソースのクロスバリデーション |
| B2Bの長い営業サイクル | 中 | B2Cでのトラクション実績をB2B営業に活用 |
| 個人情報保護法対応 | 低 | 統計データのみ使用（個人情報非保有）→ リスク低 |

## 7. 結論：最初の一手

**今すぐ取るべきアクションは「Web アプリ化」である。**

理由:
1. 現在のCLIは技術者以外が使えない → 市場の99%にリーチできていない
2. Webアプリ化により、B2C/B2Bの両方の市場検証が同時に可能
3. 既存のスコアリングエンジン・レポートテンプレートはそのまま転用可能
4. SEOによるオーガニック集客でCAC（顧客獲得コスト）を低く抑えられる
5. Web版のトラクションデータがB2B営業とファンドレイジングの根拠となる

**最小限のWeb MVP（2ヶ月）**:
- 都市名入力 → スコア比較 → 簡易Webレポート表示
- ユーザー登録 → 月3件無料 → 有料プランへの導線
- これだけで市場の反応を測定できる
