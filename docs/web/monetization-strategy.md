# マネタイズ戦略書

## エグゼクティブサマリー

TownLens は、政府統計（e-Stat）と不動産情報ライブラリAPIを統合し、市区町村・メッシュ・駅圏の多角的比較レポートを自動生成するプロダクトである。現在CLIツールとして高い完成度を持つが、ビジネスとしてのマネタイズには**プロダクト形態の転換**と**ターゲット市場の明確化**が必要となる。

本戦略では、3つのフェーズに分けて段階的にマネタイズを実現するロードマップを提案する。

## 1. 市場分析

### 1.1 ターゲット市場

| セグメント | 市場規模感 | ペインポイント | 支払意思 |
|-----------|-----------|--------------|---------|
| **引越し検討者（個人）** | 年間500万件以上の引越し | 街選びの判断材料が散在・比較が困難 | 低〜中（月額500〜2,000円） |
| **不動産仲介業者** | 約12万事業所 | 顧客への提案資料作成が手作業 | 中〜高（月額5,000〜30,000円） |
| **不動産デベロッパー** | 大手〜中堅100社超 | 用地選定・マーケティングに統計分析が必要 | 高（月額50,000〜300,000円） |
| **自治体・シンクタンク** | 約1,700自治体 | 移住促進・政策立案の根拠データ | 高（年額100万円〜） |
| **メディア・ライター** | 不動産系メディア多数 | 街ランキング記事のデータ根拠 | 中（記事単位 or 月額） |

### 1.2 競合分析

| 競合 | 強み | 弱み | 当プロダクトの差別化 |
|------|------|------|-------------------|
| SUUMO 街診断 | 圧倒的ブランド・集客力 | 主観的口コミ中心、定量データ不足 | **政府統計ベースの客観スコアリング** |
| HOME'S まちむす | UI/UX の良さ | 比較機能が限定的 | **多指標×重み付けの柔軟な比較** |
| TownU (三井不動産) | 大手のリソース | 自社物件への誘導バイアス | **中立的・第三者的な分析** |
| 各自治体オープンデータ | 無料・一次データ | 散在・フォーマット不統一・比較不能 | **統合・正規化・レポート自動生成** |

#### 間接競合

| 競合 | 概要 | 脅威度 | 当プロダクトの差別化 |
|------|------|--------|-------------------|
| RESAS（地域経済分析システム） | 政府公式の地域分析ツール。産業・人口・観光等を可視化 | 中 | RESASは政策立案向けで「住みやすさ」比較に特化していない。当プロダクトは**生活者目線のスコアリング**に特化 |
| jSTAT MAP | e-Statデータを地図上で可視化する政府ツール | 低〜中 | 地図表示のみで**多指標の統合スコアリング・レポート生成機能がない** |
| AI検索（ChatGPT / Perplexity等） | 「○○区と△△区を比較して」とAIに質問するユーザー層 | 中〜高 | AIの回答はデータの根拠が不明瞭。当プロダクトは**政府統計に基づく定量的根拠**を提供し、PDFとして保存・共有可能 |
| マンションマーケット等の不動産テック | 価格推定＋エリア情報を提供 | 低 | 不動産価格が主軸で、生活環境（犯罪・医療・教育等）の**多角的比較**は弱い |

### 1.3 当プロダクトの核心的価値（モート）

#### 初期の差別化要因（技術的価値）

1. **データ統合力**: e-Stat + 不動産情報ライブラリ + 災害データを1つのスコアに統合
2. **スコアリングエンジン**: 正規化・パーセンタイル・重みプリセット・信頼度評価の独自ロジック
3. **粒度の柔軟性**: 市区町村 / 1kmメッシュ / 駅圏 の3階層で分析可能
4. **レポート自動生成**: プロフェッショナルなPDFを完全自動生成

> **注意**: 上記の技術的価値はいずれもオープンAPI＋公知の手法に基づいており、十分な開発リソースがあれば模倣可能。そのため、以下の持続的モートの構築が不可欠。

#### 持続的モート（構築が必要）

5. **ユーザー行動データの蓄積**: どの街がどの街と比較されるか、どの指標が重視されるかの行動データ。蓄積されるほどレコメンデーションやトレンド分析の精度が向上
6. **B2Bパートナーネットワーク**: 不動産仲介会社の業務フローに組み込まれることで高いスイッチングコストを生む
7. **ブランド認知**: 「街比較＝○○」という第一想起の獲得。SEOドメインパワーの蓄積

## 5. 戦略的優先度マトリクス

```
            高収益ポテンシャル
                 │
    [B2B SaaS]   │  [自治体PKG]
    すぐ始められる │  予算サイクル待ち
                 │
速い実現 ────────┼──────── 遅い実現
                 │
    [B2Cフリー   │  [APIプラット
     ミアム]     │   フォーム]
    市場検証      │  ユーザー基盤必要
                 │
            低収益ポテンシャル
```

**推奨優先順位**:
1. **B2C フリーミアム**（速い実現・市場検証）→ まず始めるべき。全施策の基盤となる
2. **B2B SaaS**（速い実現・高収益）→ B2Cのトラクションを活用して営業。最大の収益ドライバー
3. **自治体パッケージ**（遅い実現・高収益）→ 年度予算サイクルに合わせた提案が必要。M12〜から種まき
4. **API プラットフォーム**（遅い実現・中〜高リターン）→ B2B顧客基盤が確立してからのスケール手段

| e-Stat APIの商用利用 | 高 | 利用規約上、政府統計データは自由に二次利用可能（出典明記が必要） | 商用SaaSとしての利用について総務省に事前確認を推奨 |
| 不動産情報ライブラリAPIの商用利用 | 高 | 現時点で商用利用の可否が未確認 | **ローンチ前に必ず利用規約の確認と問い合わせが必要** |
| 犯罪統計のランキング化 | 中 | 犯罪件数の比較は自治体からの風評被害クレームのリスクあり | 絶対数ではなく人口比での表示、「安全性スコア」等のポジティブな表現に変換 |
| スコアリング結果の誤解 | 中 | ユーザーがスコアを絶対的な評価として受け取る可能性 | 免責事項の明示、スコアの算出根拠と限界の透明化 |
| 個人情報保護法 | 低 | 統計データのみ使用（個人情報非保有） | ユーザーアカウント情報の取扱いについてプライバシーポリシー整備 |

## 7. 結論：最初の一手

**今すぐ取るべきアクションは「Web アプリ化」である。**

理由:
1. 現在のCLIは技術者以外が使えない → 市場の99%にリーチできていない
2. Webアプリ化により、B2C/B2Bの両方の市場検証が同時に可能
3. 既存のスコアリングエンジン・レポートテンプレートはそのまま転用可能
4. SEOによるオーガニック集客でCAC（顧客獲得コスト）を低く抑えられる
5. Web版のトラクションデータがB2B営業とファンドレイジングの根拠となる

**最小限のWeb MVP（2ヶ月）**:
- 都市名入力 → スコア比較 → 簡易Webレポート表示
- ユーザー登録 → 月3件無料 → 有料プランへの導線
- これだけで市場の反応を測定できる
