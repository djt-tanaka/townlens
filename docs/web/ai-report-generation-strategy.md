## 1. 背景と課題

### 現状のナラティブ生成の限界

現在の比較レポートのナラティブ（`packages/core/src/narrative/`）は100%ルールベースのテンプレートで、以下の構成で生成される:

| 文 | 内容 | 例 |
|----|------|-----|
| 比較: 結論文 | 1位 vs 2位の差分を記述 | "世田谷区が総合1位、渋谷区との差は5.2点" |
| 比較: トレードオフ文 | 各指標のリーダーを列挙 | "世田谷区は保育施設数で優位、渋谷区は駅数で優位" |
| 比較: プリセット影響文 | 重み配分の影響を解説 | "子育てプリセットではこの分野での優位が反映" |
| 都市別: 総合評価文 | スコアと順位 | "世田谷区は★★★★で候補3市区町村中1位" |
| 都市別: 強み・弱み文 | 閾値ベースの分類 | "保育施設数が強み。住宅価格は課題" |
| 都市別: 信頼度注記 | データ欠損の警告 | "2件の指標データが欠損" |

**問題点**: これは「データの読み上げ」であって「分析」ではない。表やグラフを見れば自明な情報しか含まれておらず、ナラティブの付加価値がほぼゼロ。ChatGPTに同じデータを渡せば同等の文章が生成可能であり、差別化要因にならない。

### マネタイズへの影響

- Free/Standard間の差別化が「量の壁」（月間レポート数）のみに依存
- テンプレナラティブの品質では「お金を払ってまで読みたい」動機が生まれない
- B2B（不動産仲介 ¥19,800/月）の提案書としてはクオリティ不足

## 2. AIナラティブの戦略的位置づけ

### 2.1 ポジショニングの転換

```
現在:  データの統合・可視化ツール       → ¥980/月（「見るだけ」）
目標:  データに基づく意思決定支援サービス → ¥2,980+/月（「教えてくれる」）
B2B:   営業提案書の自動生成機            → ¥19,800+/月（「稼がせてくれる」）
```

### 2.2 AIが生成すべきレイヤー

テンプレートが担う「事実の要約」の上に、AIが「解釈・提案」のレイヤーを追加する:

| レイヤー | 担当 | 例 |
|---------|------|-----|
| 事実の要約 | テンプレート（既存） | "犯罪率が低い" |
| 文脈の解釈 | **AI** | "隣接する○○区と比べても治安が良く、夜間の一人歩きも安心" |
| トレードオフ分析 | **AI** | "月々の住居費差額は約3万円。安全面の差を考慮すると..." |
| ペルソナ別提案 | **AI（将来）** | "共働き世帯なら保育園の充実度も要チェック" |
| アクション提案 | **AI（将来）** | "まず○○エリアの内見をお勧めします" |

### 2.3 二層アーキテクチャ: テンプレート + AI

```
┌─────────────────────────────────────────┐
│  レイヤー1: テンプレナラティブ（確定事実）  │
│  ・数値・順位・星レーティング              │
│  ・指標ごとの優位/劣位                    │
│  → 改変不可。常にレポートに表示            │
│  → LLM障害時のフォールバックにもなる       │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  レイヤー2: AIナラティブ（解釈・提案）     │
│  ・データの「意味」の自然言語解釈           │
│  ・トレードオフの「判断基準」提示           │
│  ・ペルソナに応じた視点切り替え（将来）     │
│                                           │
│  ❌ 数値の生成・計算は一切させない          │
│  ❌ 外部知識に基づく事実の主張をさせない    │
│  → 「AI分析」として明示ラベル付きで表示    │
│  → 有料プラン限定                         │
└─────────────────────────────────────────┘
```

## 3. 課金ティア設計

### 3.1 AIナラティブのティア別提供範囲

| 機能 | Free | Standard ¥980 | Premium ¥2,980 | B2B ¥19,800+ |
|------|------|---------------|----------------|-------------|
| データ表示（表・グラフ） | 全て | 全て | 全て | 全て |
| テンプレナラティブ | 全て | 全て | 全て | 全て |
| **AIナラティブ閲覧** | サンプルのみ（ブラー） | **月3件** | **無制限** | **無制限** |
| **ペルソナ別分析** | なし | なし | **将来追加** | **将来追加 + カスタム** |
| **アクション提案** | なし | なし | **将来追加** | **将来追加** |

### 3.2 サンプルレポートによるFree→Standard転換

**目的**: FreeユーザーにAIナラティブの品質を体験させ、Standard移行を促す

**設計**:

- ルート: `/report/sample` （専用のサンプルレポートページ）
- 都市ペア: **世田谷区 vs 渋谷区**（子育てプリセット）
  - 知名度が高く、子育て vs 利便性のトレードオフが明確
  - ターゲットユーザー（引越し検討者）の関心が高い組み合わせ
- データ・グラフ: **全て閲覧可能**（データの信頼性を示す）
- テンプレナラティブ: **表示**（現状品質のベースライン）
- AIナラティブ: **ブラー表示 + アップグレードCTA**

```
サンプルレポートのユーザー体験:
  1. ランキングやLPからサンプルレポートへ誘導
  2. スコア・グラフ・テンプレ文は普通に閲覧可能
  3. AIナラティブセクションがブラーで表示
     └─ 「AI分析でさらに深い比較を読めます」
     └─ [Standardにアップグレード ¥980/月] ボタン
  4. ブラーの裏にはダミーではなく実際のAIナラティブを配置
     └─ 「見えそうで見えない」感で興味を喚起
```

### 3.3 サンプルレポートの技術的実装方針

- サンプルレポートのAIナラティブは**事前生成してDB/静的JSONに格納**
  - FreeユーザーのアクセスごとにLLM APIを呼ばない（コスト防止）
  - ビルド時またはシード時に生成し、キャッシュ
- サンプルレポートのスコアデータも**事前計算してシード**
  - 世田谷区 vs 渋谷区の比較結果をfixture化
  - 元データ更新時（四半期）に再生成

## 4. AIナラティブ生成のアーキテクチャ

### 4.1 モジュール配置

```
packages/core/src/
├── narrative/                    # 既存テンプレナラティブ（変更なし）
│   ├── comparison-narrative.ts
│   ├── city-narrative.ts
│   ├── classify.ts
│   └── constants.ts
├── ai-narrative/                 # 新規: AIナラティブ生成
│   ├── types.ts                  # AiNarrativeInput, AiNarrativeResult 型定義
│   ├── provider.ts               # LLMプロバイダ抽象インターフェース
│   ├── providers/
│   │   ├── anthropic.ts          # Claude API実装
│   │   └── openai.ts             # OpenAI API実装
│   ├── prompt-builder.ts         # プロンプト構築（データ→プロンプト変換）
│   ├── generate.ts               # メインの生成ロジック
│   └── index.ts                  # バレルエクスポート
```

**配置の理由**: `packages/core` に置くことで CLI・Web 両方から利用可能。LLM APIキーは呼び出し側（web/cli）から注入する。

### 4.2 プロバイダ抽象化

```typescript
// packages/core/src/ai-narrative/provider.ts

/** LLMプロバイダの抽象インターフェース */
export interface AiNarrativeProvider {
  generate(input: AiNarrativePrompt): Promise<AiNarrativeResult>;
}

export interface AiNarrativePrompt {
  /** システムプロンプト（役割・制約の定義） */
  readonly system: string;
  /** ユーザープロンプト（構造化データ + 生成指示） */
  readonly user: string;
}

export interface AiNarrativeResult {
  /** 比較ナラティブ */
  readonly comparisonNarrative: string;
  /** 都市別ナラティブ（cityName → narrative） */
  readonly cityNarratives: ReadonlyMap<string, string>;
  /** メタデータ */
  readonly meta: {
    readonly model: string;
    readonly inputTokens: number;
    readonly outputTokens: number;
    readonly durationMs: number;
  };
}
```

初期実装はどちらか一方のプロバイダで開始し、必要に応じて切り替え可能にする。

### 4.3 プロンプト設計方針

#### システムプロンプト

```
あなたは日本の住宅・不動産の専門アナリストです。
政府統計データに基づいた都市比較分析を行います。

【厳守ルール】
1. 提供されたデータに含まれない数値を絶対に生成しないでください
2. 推測や外部知識に基づく事実の主張は避けてください
3. 「〜と考えられます」「〜の傾向があります」等の表現を使い、断定を避けてください
4. 各都市の分析は200〜300文字程度に収めてください
5. 比較分析は300〜500文字程度に収めてください
```

#### ユーザープロンプト（データ注入）

```
以下の都市比較データに基づいて、分析ナラティブを生成してください。

## 比較対象
- 都市A: {cityName} （総合★{starRating}/5.0, {rank}位/{totalCities}中）
- 都市B: {cityName} （総合★{starRating}/5.0, {rank}位/{totalCities}中）

## スコアデータ
{各指標のスコア・生データをJSON形式で注入}

## テンプレ分析（参考）
{既存テンプレナラティブの出力を参考情報として渡す}

## 出力形式
JSON形式で以下を出力:
{
  "comparisonNarrative": "比較分析テキスト（300-500文字）",
  "cityNarratives": {
    "{cityNameA}": "都市A分析テキスト（200-300文字）",
    "{cityNameB}": "都市B分析テキスト（200-300文字）"
  }
}
```

### 4.4 安全策

| リスク | 対策 |
|--------|------|
| ハルシネーション | 数値生成を禁止。テンプレナラティブを「事実層」として併記 |
| 品質のばらつき | 出力の文字数・フォーマット検証。NGワード検出 |
| APIダウンタイム | テンプレナラティブをフォールバックとして常に保持 |
| コスト急増 | レート制限（プラン別月間回数制限）。サンプルは事前生成 |
| 法的リスク | 「AI生成」の明示ラベル表示 + 免責事項 |

### 5.3 表示の判定ロジック

```typescript
// レポート表示時の判定
const isAiNarrativeVisible = (
  userPlan: 'free' | 'standard' | 'premium',
  reportType: 'sample' | 'user-generated',
  monthlyAiUsage: number,
): boolean => {
  // サンプルレポート → 常にロック（ブラー表示）
  if (reportType === 'sample') return false;

  // Free → 常にロック
  if (userPlan === 'free') return false;

  // Standard → 月3件まで
  if (userPlan === 'standard') return monthlyAiUsage < 3;

  // Premium → 無制限
  return true;
};
```
